## Summary
Dining Memory は、撮影して保存し、一覧・検索・簡易統計で振り返るところまでが動く段階にあります。  
AI 解析、クラウドバックアップ、データエクスポートはまだ実装していません。

## 現在使える機能
- カメラで写真を撮影し、内容を確認してから保存する
- 料理名、場所、メモ、自炊フラグを付けて記録する
- 記録一覧を日付ごとに見返す
- 記録を編集・削除する
- テキスト検索、場所フィルター、自炊フィルターで探す
- 総記録数、自炊比率、よく記録する料理ジャンル・場所を確認する
- 設定画面でプライバシー方針を確認し、ローカルデータを削除する

## 機能ごとの進捗
### 完了
- 撮影から保存までの基本フロー
- 記録一覧の表示、編集、削除
- 基本検索
- 簡易統計
- 設定画面の最小構成
- ローカル保存を前提にしたデータアクセス層

### 一部実装
- 統計
  - 現在は軽量なサマリーのみ
- 検索
  - 現在はテキスト、場所、自炊フィルターのみ
- 画像表示
  - サムネイル生成は未実装で、保存画像のフォールバック表示を使う

### 未実装
- AI 解析
- セマンティック検索
- クラウドバックアップ
- データエクスポート / 復元
- 高度な行動パターン分析
- 栄養情報推定
- レシピ提案

## 現在の実装範囲
### Camera
- 写真撮影
- 撮影後レビュー
- 手動入力
- 保存完了メッセージ

### Records
- 日付グループ表示
- 編集 / 削除
- 保存画像の表示

### Search
- テキスト検索
- 場所フィルター
- 自炊のみフィルター

### Stats
- 総記録数
- 自炊 / 外食件数
- 自炊比率
- よく記録する料理ジャンル・場所

### Settings
- プライバシー説明
- 現在の機能範囲の説明
- ローカルデータ削除
- アプリ情報

## 未実装 / 構想中
- ローカル AI による料理認識
- クラウド API を使った補完解析
- 意味ベース検索
- バックアップ、エクスポート、復元
- より深い統計・傾向分析

## 次に進める候補
- AI 解析を入れる前に、記録項目と検索体験の基盤を固める
- 統計をもう少し実用的にする
- バックアップとエクスポートの優先順位を決める
- 記録作成フローの入力補助を増やす
